MY ROLE

I'm a UX designer and responsible for doing user research, empathic desigins and UI design

PROJECT DESCRIPTION

An app created for people who love flowers but don’t want to spend a lot of money on a beautiful custom arrangement.

GOALS

It fills the mid-level white space between grocery store flowers and flower delivery websites. Allows users the convenience & safety of ordering flowers from home.





RESEARCH QUESTION

In this research project, We began by conducting user 6 interviews and an online survey in order to discover the factors that how people in daily life to purchase flower? Why they buy flowers? How often?



INTERVIEW TRANSCRIPT





PARTICIPANT 1

“I think they need to be organized better, they feel outdated. They need a more elevated selection. The more elevated flower delivery websites are too expensive. Too expensive, not that pretty. Too limited of a selection of bouquets.”

PARTICIPANT2

“I like: that you can select a specific window of time for delivery, I like that there's a large assortment of bouquets & the option to choose small, medium, or large for each I dislike: it doesn't feel personal, feels generic & not as thoughtful as other options, expensive, inconsistent supply chain, so doesn't feel confident the flowers will arrive as the image because of this”

PROBLEM STATEMENT

Many people forget important dates such as anniversaries, birthdays, or holidays which can cause loved ones to feel neglected or forgotten. A flower delivery calendar that allows users to input important dates for loved ones and set up automatic flower deliveries personalized to each recipient would ensure their loved ones feel appreciated even if they’re too busy to remember to send flowers on important dates.

VALUE PROPOSITION STATEMENT

Value proposition statement: Our online flower delivery platform is developing the calendar of important dates, contacts, comparing the price difference of flowers studio from nearby and subscriptions feature to help customers save money and notice them send gifts to a particular person on some important dates. We’re better because we provide better service at an affordable price.

GUERRILLA USER TEST PLAN

For the user test, we conducted 6 tests using XD wireframe prototype. All tests have scripts. I send every user 4 tasks file. They follow up each step and then give me feedback about “One Bouquett” app. All tests have been recorded.

User test tasks:
1.knows how to sign up for the app
2.knows how to reset password
3.knows how to subscribe the monthly flower delivery
4.know how to customize bouquet





AFFINITY DIAGRAM

User test issue:
1. All users understood how to use this app and where to go.Easy to use, well-navigated
2. Two user feels that the “choose a gift” option is a little bit confused
3. Can add more categories, such as recommendations by holidays.
4. It is better that users can purchase flowers directly at the calendar







USER TEST ANALYSIS

According to the feedback, We will make some improvement on prototype,making every button work. At calendar page, we add a button. When users go to check important day, they can book the flowers directly. We will think about go back process and try to make every page can go back. Last, we will make some improvement on UI design.
